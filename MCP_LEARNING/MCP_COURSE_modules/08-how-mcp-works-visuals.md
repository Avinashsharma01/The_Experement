### What Each Step Does:

| Step | What Happens | Who Does It |
|------|-------------|-------------|
| в‘  | You type a natural language question | You |
| в‘Ў | The LLM analyzes your request and decides which tool to call | Copilot (AI) |
| в‘ў | Copilot sends a structured tool call with parameters | Copilot вҶ’ MCP Client |
| в‘Ј | VS Code sends a JSON-RPC message over stdio/HTTP to the server | MCP Client вҶ’ MCP Server |
| в‘Ө | The MCP server calls the real external API (GitHub, DB, etc.) | MCP Server вҶ’ External |
| в‘Ҙ | The external service returns raw data | External вҶ’ MCP Server |
| в‘Ұ | The server packages the data and sends it back | MCP Server вҶ’ MCP Client |
| в‘§ | VS Code delivers the tool result to the AI | MCP Client вҶ’ Copilot |
| в‘Ё | Copilot turns raw data into a human-readable answer | Copilot вҶ’ You |

## рҹ”‘ The JSON-RPC Protocol вҖ” What's in the Messages?

MCP uses **JSON-RPC 2.0** under the hood. Here's what the actual messages look like:

### Step 1: Client asks "What tools do you have?"

```json
// вҶ’ Request (Client to Server)
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/list",
  "params": {}
}
```

### Step 2: Server responds with its tools

```json
// вҶҗ Response (Server to Client)
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "tools": [
      {
        "name": "list_issues",
        "description": "List issues in a GitHub repository",
        "inputSchema": {
          "type": "object",
          "properties": {
            "owner":  { "type": "string", "description": "Repo owner" },
            "repo":   { "type": "string", "description": "Repo name" },
            "state":  { "type": "string", "enum": ["open", "closed", "all"] }
          },
          "required": ["owner", "repo"]
        }
      }
    ]
  }
}
```

### Step 3: Client calls a tool

```json
// вҶ’ Request (Client to Server)
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tools/call",
  "params": {
    "name": "list_issues",
    "arguments": {
      "owner": "avinash",
      "repo": "my-project",
      "state": "open"
    }
  }
}
```

### Step 4: Server returns the result

```json
// вҶҗ Response (Server to Client)
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "Found 3 open issues:\n1. #42 - Fix login bug\n2. #58 - Add dark mode\n3. #71 - Update docs"
      }
    ]
  }
}
```

> рҹ”‘ **You never write this JSON yourself!** VS Code and the MCP server handle it automatically. But understanding the format helps you debug when things go wrong.
